1. Het Admin account

Het Admin account (tier: admin) is het hoogste account type binnen MyFamTreeCollab. Het geeft volledige toegang tot alle functies van de applicatie, inclusief het admin dashboard.

Activatie

Admin accounts worden handmatig toegewezen via de Supabase database — rechtstreeks in de profiles tabel, kolom tier. Er is geen zelfregistratie of donatie-flow voor Admin.

DB-waarde: profiles.tier = 'admin'
Toewijzing via Supabase dashboard: Table Editor → profiles → tier → admin

Verschil met Owner

  • Owner heeft alle gebruikersfuncties (cloud, samenwerken, versiebeheer max. 5)
  • Admin heeft alle Owner-functies plus toegang tot het admin dashboard
  • Admin heeft onbeperkt versiebeheer (Owner is beperkt tot 5 versies)
  • Admin kan importeren, net als Owner

3. Admin dashboard

Het admin dashboard is exclusief toegankelijk voor accounts met tier: admin. Het biedt een overzicht van platformactiviteit en gebruikersstatistieken.

Toegang: Navigeer naar /MyFamTreeCollab/admin/analytics.html of gebruik de link in het gebruikersmenu (zichtbaar alleen voor admin-accounts).

Functies in het dashboard

  • Paginabezoeken per dag / per pagina
  • Actieve gebruikers (op basis van siteAnalytics)
  • Account type verdeling (guest / owner / admin)
  • Samenwerking statistieken (gedeelde stambomen, berichten)
Beveiliging: Het dashboard controleert profiles.tier === 'admin' bij laden. Niet-admin accounts zien een toegangsgeweigerd melding en worden doorgestuurd naar de homepagina.

4. Importeren als Admin

Net als Owner kan Admin stamboomdata importeren via een CSV-bestand. Zelfde flow als de gebruikershandleiding sectie 11.

  • Download het CSV-sjabloon.
  • Vul het sjabloon in met de familiegegevens.
  • Ga naar Importeren via de navigatie.
  • Klik op "Bestand kiezen" en kies de CSV.
  • Klik op "Importeren".
Importeren is ook beschikbaar voor Owner-accounts. Zie de gebruikershandleiding sectie 11.

5. Versiebeheer (onbeperkt)

Admin heeft onbeperkt versiebeheer — geen maximum van 5 versies zoals bij Owner. Alle versies van alle stambomen blijven beschikbaar.

  • Ga naar Mijn Account → 📜 Versiegeschiedenis.
  • Kies een stamboom uit de dropdown — alle versies worden getoond.
  • Klik op "👁 Preview" om een versie te bekijken.
  • Klik op "⏪ Terugzetten" om een versie te herstellen.
  • Klik twee keer op "🔍 Vergelijk" voor een zij-aan-zij vergelijking.
Verschil met Owner: Owner is beperkt tot max. 5 versies per stamboom. Oudere versies worden automatisch verwijderd bij de Owner. Bij Admin blijven alle versies bewaard.

6. Stambomen & samenwerking

Admin heeft dezelfde stamboom- en samenwerkingsfuncties als Owner: cloud backup, stamboom delen, Viewer/Editor uitnodigen en het berichtenboard modereren.

Zie de gebruikershandleiding sectie 10 (Cloud stambomen) en sectie 14 (Stamboom delen) voor de volledige instructies.

Aanvullende admin-bevoegdheden (toekomstig)

De volgende functies zijn voorzien voor toekomstige admin-uitbreidingen:

  • Alle gebruikersaccounts beheren (tier wijzigen, account verwijderen)
  • Alle gedeelde stambomen inzien en modereren
  • Platform-brede notificaties versturen
Status: Bovenstaande functies zijn nog niet geïmplementeerd (backlog). Huidig admin-voordeel ten opzichte van Owner: dashboard + onbeperkt versiebeheer + importeren.
